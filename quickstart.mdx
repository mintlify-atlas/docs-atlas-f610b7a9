---
title: "Get started with Kashio"
description: "Learn how to install Kashio, configure your environment, run the database migrations, and log your first AI-parsed expense in minutes."
---

This page walks you from zero to a working Kashio instance with a logged expense. By the end you'll have the app running locally, a database ready to go, and a parsed expense saved to your account.

<Steps>
  <Step title="Prerequisites">
    Make sure you have the following installed and available before continuing:

    - **Node.js 20+** — required by Next.js
    - **Bun 1.x** — used for dependency installation and helper scripts
    - **PostgreSQL** — Kashio stores all data in a PostgreSQL database
    - **Groq API key** — used for AI expense parsing and voice transcription

    <Note>
      Bun is recommended over npm for this project. Some helper scripts (`seed:demo`, `db:reset`, `reset:demo`) run through Bun directly and will not work with npm.
    </Note>
  </Step>

  <Step title="Clone and install dependencies">
    Clone the repository and install dependencies:

    <CodeGroup>

    ```bash bun (recommended)
    git clone https://github.com/lifeashaze/kashio.git
    cd kashio
    bun install
    ```

    ```bash npm
    git clone https://github.com/lifeashaze/kashio.git
    cd kashio
    npm install
    ```

    </CodeGroup>
  </Step>

  <Step title="Configure environment variables">
    Create a `.env.local` file in the project root with the following content:

    ```bash .env.local
    DATABASE_URL="postgresql://postgres:postgres@localhost:5432/kashio"
    BETTER_AUTH_SECRET="replace-with-a-random-secret"
    BETTER_AUTH_URL="http://localhost:3000"
    GROQ_API_KEY="your-groq-api-key"
    RESEND_API_KEY=""
    AUTH_FROM_EMAIL="Kashio <auth@example.com>"

    # Optional OAuth
    GITHUB_CLIENT_ID=""
    GITHUB_CLIENT_SECRET=""
    GOOGLE_CLIENT_ID=""
    GOOGLE_CLIENT_SECRET=""

    # Optional Telegram integration
    TELEGRAM_BOT_TOKEN=""
    TELEGRAM_BOT_USERNAME=""
    TELEGRAM_WEBHOOK_SECRET=""
    APP_BASE_URL="http://localhost:3000"
    ```

    The four required variables to get started are `DATABASE_URL`, `BETTER_AUTH_SECRET`, `BETTER_AUTH_URL`, and `GROQ_API_KEY`. All other variables are optional and can be left empty for local development.

    See the [environment variables reference](/environment-variables) for a full description of every variable.

    <Warning>
      The `telegram:webhook:set` script reads environment variables from the shell or a `.env` file — not from `.env.local`. If you keep your Telegram secrets only in `.env.local`, export them to the shell first or duplicate them into a `.env` file before running that script.
    </Warning>
  </Step>

  <Step title="Initialize the database">
    Run the database migrations to create all required tables:

    ```bash
    bun run db:migrate
    ```

    During development, you can skip migration files and push the schema directly instead:

    ```bash
    bun run db:push
    ```

    Use `db:push` for quick local iteration when you don't need a migration history. Use `db:migrate` when you want to apply the checked-in migration files exactly as they are.
  </Step>

  <Step title="Start the app">
    Start the local development server:

    ```bash
    bun run dev
    ```

    Open `http://localhost:3000` in your browser. You should see the Kashio landing page.

    <Tip>
      Want to explore the app with realistic data before entering your own expenses? Run `bun run seed:demo` after completing onboarding to create a demo account (`demo@kashio.app` / `demo1234!`) pre-loaded with roughly six months of expenses.
    </Tip>
  </Step>

  <Step title="Complete onboarding">
    Sign up for a new account. After signing up, Kashio takes you through a two-step onboarding wizard:

    1. **Budget and currency** — Set your monthly budget amount and preferred currency.
    2. **Categories** — Choose which expense categories you want to track (food, transport, entertainment, shopping, bills, health, groceries, travel, education, and others).

    Once you complete the wizard you'll land on the home dashboard.
  </Step>

  <Step title="Log your first expense">
    On the home dashboard, find the expense input at the top of the page. Type a plain-English description of an expense:

    ```
    lunch $12
    ```

    Kashio sends this to its AI parser (via Groq) to extract the amount, category, description, and date. If the parse has high confidence, the expense is saved automatically and appears in your expense list. If the AI is less certain — for example, if a field is missing or ambiguous — a confirmation dialog opens so you can review and adjust the parsed values before saving.

    Try other formats to see how the parser handles them:

    ```
    uber to airport $42 yesterday
    coffee at the bookshop $6
    bought groceries $55.40
    ```
  </Step>
</Steps>
